Documentation

Included in the download for the .make build system is an example Makefile and a '.make' folder. We'll refer to this example Makefile as a .make Makefile. Think of it as the public interface to the .make library, whose functionity is located in the '.make' folder (a collection of GNU Makefiles).

To use the .make build system, copy the '.make' folder (and all of its contents) into your project, along with the example .make Makefile. The .make Makefile does not need to be located in the same directory as the '.make' folder (see PROJECT_BASE_DIR for more info), but in most simple projects, it will be. See the examples section for an example project setup.

Makefile Configuration Options

A .make Makefile acts as the public interface to the internal '.make' library. It contains a series of public, project-specific configuration options which you may need to edit, depending on your project's needs. Note that most simple projects will have no need to edit these options since the defaults are meant to be as all encompassing and generic as possible.

Within a .make Makefile, all public variables are prefixed by 'PROJECT_' and, by default, they are all commented out with lines beginning with '##' (in GNU Makefiles, a '#' pound sign denotes the start of a comment).
